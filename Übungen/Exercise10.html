10.1. Static classes in TypeScript

Gibt es in TypeScript statische Klassen? Erläutern Sie den Unterschied von TypeScript zu anderen objekt-orientierten, statisch typisierten Programmiersprachen wie Java und C#.

Antwort: Ja, TypeScript unterstützt statische Klassen; TypeScript ist eine statisch typisierte Sprache, wie Java und C#, aber sie erlaubt optionale Typ-Annotationen und hat eine automatische Typ-Inferenz, was die Verwendung im Vergleich zu diesen anderen Sprachen erleichtert.

10.2. Fehler finden mit TypeScript

Gegeben sei folgendes fehlerbehaftete JavaScript-Programm. Ergänzen Sie dieses um möglichst viele Typisierungen und übersetzen Sie es in ein TypeScript-Programm. 

      const add = (x,y) => x+y;
      const equals = (x,y) => x===y;
      
      console.log( add(1,2) );
      console.log( add(true, true ) );
      console.log( add(true, false ) );
      var x = y = 1;
      console.log( add(equals(x,y), equals(y,x)) );
Was ist der Fehler in dem gegebenen JavaScript-Code? Welche Fehler meldet TypeScript? Geben Sie weitere Beispiele an, wie TypeScript bei der Fehlersuche helfen kann.

Antwort:

        const add = (x: number, y: number): number => x + y;
        const equals = (x: any, y: any): boolean => x === y;
        
        console.log(add(1, 2));
        console.log(add(true, true));
        console.log(add(true, false));
        let x = 1;
        let y = 1;
        console.log(add(equals(x, y), equals(y, x)));
einige Fehler, die ich in dem Javascript-Code gefunden habe: Die Funktion add soll zwei Argumente desselben Typs annehmen, aber sie kann Argumente beliebigen Typs annehmen. Gleiches gilt für die Funktion equals; Die Variablenzuweisung var x = y = 1 ist ein Syntaxfehler
Im TypeScript-Programm werden die Fehler im JavaScript-Code vom TypeScript-Compiler als Typ-Fehlanpassungen gemeldet. Dies ermöglicht es Entwicklern, diese Fehler frühzeitig zu erkennen
